# 常见问题

## 通用问题

??? question "我的数据会存储在外部服务器上吗？"
    不会。所有内容都在您的服务器上本地运行。您的简历数据存储在 `/data` 目录中的 SQLite 数据库文件中。

??? question "可以不用 Docker 运行 CV Manager 吗？"
    可以。安装 Node.js 18+，在项目目录中运行 `npm install`，然后执行 `node src/server.js`。管理端运行在端口 3000，公开站点运行在端口 3001。

??? question "多人可以使用同一个实例吗？"
    CV Manager 设计为单用户应用程序。每个实例管理一个人的简历。如果需要多人使用，请运行独立的容器。

## 编辑

??? question "如何将一个职位标记为'在职'？"
    将**结束日期**字段留空。它将在简历上显示为"至今"。

??? question "可以重新排序板块中的条目吗？"
    可以。大多数条目支持拖拽排序。排序顺序会自动保存。

??? question "如何为工作经历添加要点？"
    编辑该工作经历，在**要点描述**字段中输入内容 — 每行一个要点。

??? question "如何添加公司标志？"
    编辑该工作经历，滚动到**公司标志**部分，点击**选择图片**上传。您也可以点击**使用已有标志**来复用之前上传的标志。启用**"将标志同步到所有 [公司名] 的经历"**开关，可将同一标志应用于该公司的所有工作经历。

??? question "我不小心删除了内容。可以撤销吗？"
    没有撤销功能。由于编辑会自动保存到活动数据集，更改会立即持久化。如果您有之前的导出文件或单独保存的数据集，可以从中恢复。定期导出简历作为备份是一个好习惯。

## 自定义板块

??? question "可以创建多少个自定义板块？"
    没有数量限制。您可以根据需要创建任意数量。

??? question "创建后可以更改自定义板块的布局类型吗？"
    可以。编辑该板块并选择不同的布局。请注意，某些字段在布局类型之间可能无法迁移（例如，从卡片切换到社交链接）。

??? question "'要点列表'和'自由文本'布局有什么区别？"
    **要点列表**将每行渲染为带有组标题的项目符号列表项。**自由文本**渲染保留换行的纯文本，没有标题 — 类似于关于我/简介板块。

## 打印与 PDF

??? question "为什么我的 PDF 看起来和屏幕上不一样？"
    打印输出使用针对纸张优化的专用打印样式。某些视觉效果（悬停状态、动画、渐变）会被简化。隐藏的条目和管理控件会自动移除。

??? question "如何让简历缩减到更少的页数？"
    尝试在打印与导出设置中启用**允许板块分割**和**允许条目分割**。您还可以隐藏不太重要的条目或板块，或使用更紧凑的自定义板块布局。也可以通过任何浏览器的打印模态窗口调整打印缩放比例（有时这个选项有点隐蔽）。

??? question "为什么打印的简历中缺少某些条目？"
    检查这些条目是否已被切换为隐藏（眼睛图标）。隐藏的条目会从打印输出和公开视图中排除。

??? question "页码没有显示？"
    确保在设置 → 打印与导出中启用了**页码**。某些浏览器 PDF 查看器可能不显示 CSS 生成的页码 — 尝试下载 PDF 并在专用阅读器中打开。

## 时间线

??? question "时间线显示的日期不对/只显示年份/显示完整日期？"
    时间线有自己的日期设置。前往**设置 → 高级 → 时间线：仅显示年份**在仅年份显示和完整日期格式之间切换。

??? question "可以直接向时间线添加条目吗？"
    不可以。时间线根据您的工作经历自动生成。添加或编辑工作经历，时间线会相应更新。

??? question "时间线上的国旗没有显示？"
    确保工作经历上的**国家代码**字段设置为有效的两位字母 ISO 国家代码（例如 `us`、`gb`、`ch`、`de`、`fr`）。国旗从外部 CDN 加载。

??? question "同时有两份工作会怎样？"
    时间线会自动检测重叠的职位并将它们渲染为**并行轨道**。同时进行的工作出现在抬升的分支线上，S 形曲线连接器显示分叉和合并点。无需配置 — 完全基于您的起止日期。短于 1 个月的重叠会被忽略（在工作交接期间很常见）。

??? question "为什么时间线显示标志而不是公司名称？"
    如果您为该工作经历上传了公司标志，时间线会显示标志图片而不是文字。如果标志文件缺失，会回退到公司名称。要从时间线移除标志，请编辑该工作经历并在公司标志部分点击**删除**。

## 语言与更新

??? question "如何更改管理端语言？"
    点击工具栏中的**地球图标**，从下拉网格中选择一种语言。更改立即生效并跨会话保存。

??? question "如何查看当前运行的版本？"
    打开**设置** — 版本号显示在模态窗口的左下角（例如 `v1.11.0`）。

??? question "即使有新版本发布，我也看不到更新横幅？"
    版本检查结果缓存 24 小时。重启服务器（或 Docker 容器）可以清除缓存并强制重新检查。您的服务器还需要出站互联网访问权限以连接 `raw.githubusercontent.com`。

## 数据集 / 多版本简历

??? question "什么是'默认'数据集？"
    默认数据集是访客在您的根 URL（`/`）看到的简历版本。首次安装时，CV Manager 会自动从您的简历数据创建一个"默认"数据集。您可以随时使用打开模态窗口中的单选按钮更改哪个数据集为默认。

??? question "我的编辑会自动保存吗？"
    是的。您在管理端所做的每项更改（添加、编辑、删除、重新排序、切换可见性）在短暂延迟后会自动保存回活动数据集。横幅会显示"保存中..."然后显示"已保存"来确认。

??? question "'加载'数据集会怎样？"
    加载数据集会将您的工作副本切换到该数据集。您之前的编辑已经自动保存，因此不会丢失。

??? question "访客可以实时看到我的编辑吗？"
    不可以。公开站点提供的是冻结的默认数据集，而不是您的实时编辑。访客只有在自动保存将更改写入默认数据集后才能看到更改。如果您正在编辑非默认数据集，访客在您将其设为默认之前完全看不到这些更改。

??? question "访客可以看到我保存的数据集吗？"
    只有当您将其设为公开时才可以。每个数据集在打开模态窗口中都有一个开关。设为公开后，该版本可以在公开站点（端口 3001）的 `/v/slug` 路径下访问。私有数据集仅可从管理界面预览。

??? question "如何与他人分享特定的简历版本？"
    打开**打开...**模态窗口，将数据集切换为公开，然后点击 slug URL 旁边的复制图标。分享该链接 — 它在公开站点上有效，不会暴露您的管理界面。

??? question "可以同时公开多个版本吗？"
    可以。您可以将任意数量的数据集设为公开。每个都有自己的 URL（例如 `/v/technical-cv-1`、`/v/marketing-cv-2`）。主 `/` 页面显示默认数据集。

??? question "可以删除默认数据集吗？"
    不可以。当前通过单选按钮选为默认的数据集无法删除。请先将另一个数据集设为默认，然后再删除旧的数据集。

??? question "搜索引擎会索引我的版本化 URL 吗？"
    默认情况下不会 — 版本化页面带有 `noindex, nofollow` 标签。要允许索引，请在设置 → 高级中启用**索引版本化 URL**。

## 公开站点与 SEO

??? question "如何分享我的简历？"
    分享您的公开服务器（端口 3001）的 URL。如果您已使用 Cloudflare Tunnel 或反向代理设置了域名，请分享该域名。根 URL 始终显示您的默认数据集。您也可以使用公开的版本化 URL 分享特定版本（参见[数据集](../guide/datasets.md)）。

??? question "搜索引擎会索引我的简历吗？"
    默认情况下会 — 主公开页面包含正确的 meta 标签、站点地图和 robots.txt。要阻止索引，请在设置 → 高级中将**搜索引擎索引**设置更改为"No Index"。公开的版本化 URL（`/v/slug`）默认**不被索引**；如果您希望它们被抓取，请启用**索引版本化 URL**。

??? question "可以为简历添加 Google Analytics 吗？"
    可以。在**设置 → 高级 → 跟踪代码**中粘贴您的跟踪代码。它仅注入到面向公众的页面中。

## Docker 与基础设施

??? question "我的更改没有出现在公开站点上？"
    公开站点提供的是**默认数据集**，当您在管理端编辑时会自动更新。尝试在公开站点上进行硬刷新（`Ctrl+Shift+R`）。如果运行的是独立容器，请确保它们共享同一个数据卷。

??? question "出现'端口已被占用'错误？"
    更改 Docker 配置中的主机端口映射。例如，映射为 `3010:3000` 和 `3011:3001`。**不要**更改 `PUBLIC_PORT` 环境变量 — 那是容器内部端口。

??? question "如何备份我的数据？"
    两种方式：使用管理工具栏中的**导出**按钮（导出 JSON），或备份包含 SQLite 数据库和上传图片的 `data/` 目录。

??? question "个人头像没有显示？"
    确保图片是通过管理界面上传的。文件存储在 `data/uploads/picture.jpeg`。如果在 Linux 上运行，请检查文件权限。
